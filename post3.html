How to Deploy Websites on GitHub Pages

By IamZurfishan • 20 Apr 2026 • 12 min read

Want to share your website with the world for FREE? GitHub Pages is the answer! In this complete guide, I'll walk you through every step to deploy your static websites and get a live URL like https://iamZurfishan.github.io.

🎯 What is GitHub Pages?

GitHub Pages is a free hosting service that turns your GitHub repository into a live website. It's perfect for:

  • Student portfolios and lab projects
  • Personal blogs and resumes
  • Open source project documentation
  • Testing and sharing your work with anyone
💰 Best Part

It's 100% FREE! No hosting costs, no server setup, no credit card required. Just your code on GitHub.

📋 Prerequisites (What You Need)

  • A GitHub account (sign up at github.com if you don't have one)
  • Your website files (HTML, CSS, JS) ready on your computer
  • Basic knowledge of Git commands (or GitHub Desktop app)
  • Internet connection (obviously!)

🚀 Method 1: Deploy with GitHub Website (Easiest for Beginners)

Step 1: Create a New Repository

📦 Creating Repository on GitHub 
1. Log into your GitHub account
2. Click the "+" icon in top-right corner → "New repository"
3. Repository name: your-username.github.io
   (Example: iamZurfishan.github.io)
4. Make it PUBLIC (required for free hosting)
5. Check "Add a README file" (optional)
6. Click "Create repository"
⭐

Important! The repository name MUST be exactly: username.github.io (replace "username" with your actual GitHub username). This creates your main site at that URL.

Step 2: Upload Your Website Files

📤 Uploading Files 
1. In your new repository, click "Add file" → "Upload files"
2. Drag and drop ALL your website files:
   - index.html (MUST have this!)
   - style.css
   - script.js
   - post1.html, post2.html, post3.html
   - images folder (if any)
3. Add a commit message: "Initial website upload"
4. Click "Commit changes"

Step 3: Enable GitHub Pages

⚙️ Activate Pages 
1. Go to your repository's "Settings" tab
2. Scroll down to "GitHub Pages" section (or click "Pages" in left sidebar)
3. Under "Branch", select "main" (or "master")
4. Click "Save"
5. Wait 1-2 minutes for deployment
6. Your site will be live at: https://yourusername.github.io
🕐 Deployment Time

It usually takes 1-3 minutes for your site to go live after enabling Pages. Refresh the page and you'll see the green checkmark with your URL!

💻 Method 2: Deploy with Git Commands (Professional Way)

Step 1: Initialize Git Locally

🖥️ Terminal Commands 
# Navigate to your project folder
cd path/to/your-website-folder

# Initialize Git repository
git init

# Add all files
git add .

# Make first commit
git commit -m "Initial commit - My awesome website"

Step 2: Connect to GitHub Remote

🔗 Link Local to Remote 
# Add remote origin (replace with your repo URL)
git remote add origin https://github.com/IamZurfishan/IamZurfishan.github.io.git

# Push to GitHub
git branch -M main
git push -u origin main

Step 3: Enable Pages (Same as Method 1, Step 3)

🔄 Updating Your Live Website

When you make changes to your website, here's how to update it:

🔄 Push Updates 
# After making changes to your files
git add .
git commit -m "Update: Describe your changes here"
git push

# OR using GitHub website:
# Just upload the updated files again - it will replace them automatically
⚡

Pro Tip: Changes usually appear within 1-2 minutes. If you don't see them, do a hard refresh (Ctrl+Shift+R or Cmd+Shift+R) to clear your browser cache.

📁 Custom Domain (Optional - Use Your Own URL)

Want your site at www.yourname.com instead of username.github.io? Here's how:

🌐 Custom Domain Setup 
# Step 1: Buy a domain from Namecheap, GoDaddy, Google Domains, etc.

# Step 2: Add CNAME file to your repository
# Create a file named "CNAME" (no extension)
# Inside, write: www.yourdomain.com

# Step 3: In GitHub Pages settings, enter your custom domain

# Step 4: Configure DNS settings at your domain provider:
# - Add A records pointing to GitHub's IP addresses
# - Or add CNAME record pointing to username.github.io

🐛 Common Deployment Issues & Fixes

❌ Issue 1: "404 Page Not Found"

🔧 Solution 
✅ Make sure you have an index.html file in the root folder
✅ Check that you're using the correct URL (username.github.io)
✅ Wait 2-3 minutes after enabling Pages
✅ Check if branch is set correctly (main or master)

❌ Issue 2: CSS/JS Not Loading

🔧 Solution 
✅ Check file paths - use relative paths, not absolute
   ❌ Wrong: href="/style.css"
   ✅ Correct: href="style.css"
✅ Make sure all file names match exactly (case-sensitive!)
✅ Check browser console for 404 errors

❌ Issue 3: Images Not Showing

🔧 Solution 
✅ Check image paths: src="images/photo.jpg"
✅ Make sure images are uploaded to the repository
✅ Use supported formats: .jpg, .png, .gif, .svg, .webp
✅ Check for typos in file names

❌ Issue 4: My Site Shows Old Content

🔧 Solution 
✅ Clear your browser cache (Ctrl+Shift+Delete)
✅ Do a hard refresh (Ctrl+Shift+R / Cmd+Shift+R)
✅ Check GitHub repository - files should show recent commit
✅ Wait up to 10 minutes (sometimes GitHub takes longer)

🎯 Project Page vs User/Organization Page

GitHub Pages offers two types of sites:

📖 Two Types of Pages 
# 1. USER SITE (One per account)
Repository name: username.github.io
URL: https://username.github.io
Use for: Main portfolio, personal website

# 2. PROJECT SITE (Unlimited)
Repository name: any-name (e.g., my-blog)
URL: https://username.github.io/repository-name
Use for: Individual projects, documentation, demos
💡 My Recommendation

Create a username.github.io repository for your main portfolio, then use project sites for individual lab assignments. Both are free!

📊 Complete Deployment Checklist

  • ✅ Create GitHub account
  • ✅ Create repository (username.github.io for main site)
  • ✅ Upload all website files (must include index.html)
  • ✅ Go to Settings → Pages
  • ✅ Select branch: main (or master)
  • ✅ Click Save
  • ✅ Wait 1-3 minutes
  • ✅ Visit your live URL
  • ✅ Share with friends and professors!
🎉

Congratulations! Your website is now live on the internet. Anyone with the link can see your work. This is perfect for sharing lab projects, your portfolio, or just showing off your skills!
