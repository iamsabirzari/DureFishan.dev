Git & GitHub: Complete Beginner's Guide to Push, Pull & More

By IamZurfishan • 18 Apr 2026 • 10 min read

Version control was intimidating at first, but now I use Git every single day. In this guide, I'll walk you through all essential commands - from cloning to pushing, branching to merging. Let's make Git simple!

📌 What is Git & Why Should You Care?

Git is a version control system that tracks changes in your code. Think of it like a "save point" in video games - you can always go back to a working version if something breaks. GitHub is a cloud platform where you can store your Git repositories online.

💡 Why I Love Git

Before Git, I had folders named "final_version", "final_final", "final_final_2" - never again! Git saves me from that nightmare.

🚀 Setting Up Git on Your Computer

📥 Install Git (macOS, Windows, Linux) 
# For macOS (using Homebrew)
brew install git

# For Ubuntu/Debian Linux
sudo apt-get install git

# For Windows - Download from https://git-scm.com/

# Configure your identity (MUST DO!)
git config --global user.name "IamZurfishan"
git config --global user.email "iamZurfishan@example.com"

# Check configuration
git config --list

📁 Basic Git Commands You'll Use Every Day

1️⃣ Initialize a Repository

🏠 Starting a new project 
# Create a new folder and navigate into it
mkdir my-project
cd my-project

# Initialize Git repository
git init

# Check status (always do this!)
git status

2️⃣ Staging & Committing Changes

📝 The Commit Workflow 
# Create a file
echo "Hello World" > index.html

# Add file to staging area
git add index.html
# OR add all files
git add .

# Commit with a message
git commit -m "Add index.html file"

# View commit history
git log --oneline
✅

Pro Tip: Write meaningful commit messages! "Fix stuff" is bad. "Fix navigation bar responsiveness" is good.

🔗 Connecting to GitHub (Remote Repository)

🌍 Link Local to Remote 
# Add remote origin (replace with your repo URL)
git remote add origin https://github.com/IamZurfishan/my-project.git

# Verify remote
git remote -v

# Push to GitHub (first time)
git push -u origin main

# After first time, just use:
git push

📥 Cloning Existing Repositories

📦 Downloading Projects from GitHub 
# Clone a repository (HTTPS)
git clone https://github.com/username/repository-name.git

# Clone with SSH (more secure)
git clone git@github.com:username/repository-name.git

# Clone into specific folder
git clone https://github.com/username/repo.git my-folder

🔄 Git Pull: Getting Latest Changes

⬇️ Update Your Local Repository 
# Fetch and merge changes from remote
git pull

# Pull from specific branch
git pull origin main

# Fetch without merging (see what changed first)
git fetch
git diff origin/main
⚠️ Important!

Always run git pull before starting work to avoid merge conflicts. I learned this the hard way!

🌿 Branching: Work on Features Without Breaking Things

🌲 Branch Management 
# List all branches
git branch

# Create new branch
git branch feature-navbar

# Switch to branch
git checkout feature-navbar
# OR create and switch in one command
git checkout -b feature-navbar

# Merge branch into main
git checkout main
git merge feature-navbar

# Delete branch after merging
git branch -d feature-navbar

⚡ Essential Commands Cheat Sheet

📋 Quick Reference 
# STATUS & INFO
git status          # What's changed?
git log --oneline   # View commit history
git diff            # See unstaged changes

# UNDOING THINGS
git restore file    # Discard changes in working directory
git reset HEAD file # Unstage file
git reset --soft HEAD~1  # Undo last commit (keep changes)
git reset --hard HEAD~1  # Undo last commit (DELETE changes)

# WORKING WITH REMOTES
git remote -v       # List remotes
git push origin main # Push to GitHub
git pull origin main # Pull from GitHub
git clone url       # Copy repository

🐛 Common Git Errors & Solutions

❌ Error 1: "failed to push some refs"

🔧 Solution: Pull first! 
git pull origin main --rebase
git push origin main

❌ Error 2: "Merge conflict"

🔧 Solution: Resolve conflicts manually 
# Open conflicted files, look for <<<<<<< and >>>>>>>
# Edit to keep the correct version
git add .
git commit -m "Resolve merge conflict"
git push

❌ Error 3: "detached HEAD"

🔧 Solution: Create a branch 
git checkout -b my-new-branch
🎯

My Git Workflow (Daily Routine):

  1. git pull - Get latest changes
  2. git checkout -b feature-name - Create new branch
  3. Write code, test, repeat
  4. git add . - Stage changes
  5. git commit -m "Description" - Commit
  6. git push origin feature-name - Push branch
  7. Create Pull Request on GitHub → Merge → git pull on main

🔑 Setting Up SSH Keys (More Secure than HTTPS)

🔐 Generate and Add SSH Key 
# Generate SSH key
ssh-keygen -t ed25519 -C "iamZurfishan@example.com"

# Start SSH agent
eval "$(ssh-agent -s)"

# Add key to agent
ssh-add ~/.ssh/id_ed25519

# Copy public key
cat ~/.ssh/id_ed25519.pub

# Add to GitHub: Settings → SSH and GPG keys → New SSH key

# Test connection
ssh -T git@github.com

📊 .gitignore: What NOT to Track

🚫 Create .gitignore file 
# .gitignore example
node_modules/
.env
.DS_Store
dist/
build/
*.log
*.tmp
.vscode/
.idea/

# Create and add to Git
echo "node_modules/" >> .gitignore
git add .gitignore
git commit -m "Add .gitignore"
